Quantum error correction (QEC) is essential for fault-tolerant quantum computing — but decoding must keep pace with the hardware. Standard decoders like minimum-weight perfect matching (MWPM, implemented by PyMatching) are near-optimal under simplified noise assumptions, but real hardware noise is more complex. As part of the NVIDIA Ising open model family, a neural pre-decoder was released to bridge this gap.

What Is a Pre-Decoder?

A pre-decoder is a lightweight neural network that preprocesses syndromes before the main decoder runs. It does not replace the decoder — it makes the decoder's job easier:

Syndrome → Neural Net → Sparser Residual Syndrome → Decoder (e.g., PyMatching) → Final Decision

The neural network learns to identify and correct local, high-confidence errors from the raw syndrome. What remains — the residual — is sparser and contains only the harder, ambiguous errors. The main decoder then operates on this reduced problem, achieving both lower logical error rates and faster decode times.

In this post, we evaluate the NVIDIA Ising CNN pre-decoder on surface codes, then ask whether the same idea can generalize to quantum LDPC codes with fundamentally different structure. We implement a simple MLP pre-decoder for bivariate bicycle (BB) codes and compare results, identifying what works and where the gaps are.

Part I: CNN Pre-Decoder on Surface Codes

Architecture

The NVIDIA Ising surface code pre-decoder is a 3D CNN that operates on detector syndrome tensors of shape (B, 4, T, D, D) — four channels (X/Z syndromes + presence masks) across T measurement rounds and a D × D spatial grid. The network processes this tensor through residual 3D convolutional blocks, outputting corrected syndromes with reduced density. These are then passed to PyMatching (MWPM) for the final logical error decision.

The architecture exploits the surface code's regular lattice: each convolutional layer captures local spatiotemporal error correlations within its receptive field R = 1 + Σ(ki − 1), and the regular grid means every spatial position is treated uniformly. Two pre-trained models are available — R=9 (4 layers, ~913K params) and R=13 (6 layers, ~1.8M params), trained with a 25-parameter circuit-level noise model [3] [5].

Why Can a Pre-Decoder Beat PyMatching?

At first glance, this is counter-intuitive: PyMatching implements MWPM on the detector error model (DEM), which is near-optimal for independent errors. How can a heuristic CNN improve on that?

The key is that the pre-decoder is not replacing PyMatching — it is reducing the problem's complexity before PyMatching runs:

  1. The DEM is an approximation. PyMatching builds its matching graph by decomposing correlated circuit-level noise into independent error mechanisms via approximate_disjoint_errors. This decomposition is lossy — it discards spatiotemporal correlations from CNOT error propagation, idle depolarization, and SPAM noise. The CNN, trained end-to-end on the full 25-parameter noise model [3], learns these correlations directly within its receptive field.
  2. Division of labor. Most errors are local and fall within the CNN's receptive field (R=9 or R=13). The CNN corrects these, leaving only long-range or ambiguous errors in the residual. PyMatching then operates on a sparser, simpler matching problem — which it solves more accurately (and faster).
  3. Falsifiable. A random (untrained) CNN would add noise to the syndrome, making PyMatching worse. The fact that improvement scales with model capacity (R=13 > R=9) confirms the CNN is exploiting real structure in the noise, not benefiting from an evaluation artifact.
In short: MWPM is near-optimal given its DEM, but the DEM is an approximation of the true noise. The CNN bridges the gap.

Surface Code Limitation

These results validate neural pre-decoding on surface codes — but surface codes store just 1 logical qubit in d2 data qubits. Can we bring neural pre-decoding to codes with better encoding rates?

Part II: Bivariate Bicycle Codes

Why Move Beyond Surface Codes?

The qubit overhead of surface codes is arguably the single largest obstacle to building a useful fault-tolerant quantum computer. Current hardware roadmaps project thousands of physical qubits, but a surface-code architecture would consume most of them on error correction alone.

qLDPC codes offer a fundamentally different tradeoff [8]. Bivariate bicycle (BB) codes [1] strike an attractive balance: their quasi-cyclic structure admits relatively simple routing schemes while delivering dramatic encoding rate improvements:

The BB [[72,12,6]] code stores 12 logical qubits in 72 physical qubits — twice what a d=6 surface code uses for just 1. The [[144,12,12]] code achieves distance 12 with the same qubit count as a d=12 surface code, but encodes 12 logicals instead of 1.

Code Construction

A BB code is defined over ℤℓ × ℤm with polynomials A(x,y) and B(x,y):

HX = [A | B],   HZ = [BT | AT]

The CSS condition HXHZT = 0 follows from the commutativity of the group algebra. Construction is straightforward:

Python
# [[72, 12, 6]]: l=6, m=6
A = poly_to_matrix([(3,0), (0,1), (0,2)], 6, 6)
B = poly_to_matrix([(0,3), (1,0), (2,0)], 6, 6)
Hx = sp.hstack([A, B])      # 36 x 72
Hz = sp.hstack([B.T, A.T])  # 36 x 72

Decoding: The Degeneracy Challenge

Unlike surface codes where MWPM works well, BB codes require belief propagation (BP) [9] — but quantum degeneracy causes BP to oscillate between equivalent error configurations instead of converging [2] [6]. BP-OSD addresses this with Ordered Statistics Decoding as a fallback [7]: when BP fails, OSD uses BP's soft information to guide a deterministic search.

At p = 0.03, the [[144,12,12]] code achieves block LER ≈ 0.004 while encoding 12 logical qubits — a 12× improvement in encoding rate over a surface code at the same distance.

Part III: MLP Pre-Decoder for BB Codes

The surface code CNN exploits the regular T × D × D grid — but BB codes have non-local and non-planar check-qubit connectivity, so a CNN's local receptive field would miss long-range correlations entirely. As a first step toward neural pre-decoding for qLDPC codes, we implement a simpler architecture.

Approach

An MLP pre-decoder takes the full syndrome as input and outputs refined per-qubit error probabilities for BP-OSD:

Pipeline
Syndrome → MLP → Refined Channel Probs → BP-OSD → Correction

The output is interpolated with the uniform prior (α = 0.4) for stability. The MLP doesn't need to decode — it just provides better soft information than the flat prior p.

Note the contrast with Part I: the surface code CNN outputs corrected syndromes (a discrete residual for PyMatching), while the BB code MLP outputs refined probabilities (continuous soft information for BP-OSD). The approach differs because the downstream decoders differ.

The surface code CNN delivers consistent improvement across all noise rates because the convolutional architecture is perfectly matched to the code's lattice — every layer captures meaningful local correlations. The BB code MLP shows moderate gains at low p that vanish above threshold because it treats the syndrome as a flat vector, with no inductive bias for the code's long-range connectivity. Above threshold the MLP's predictions actually degrade performance slightly (0.95x at p=0.06).

The takeaway: syndrome-conditioned soft information can help BP-OSD, but the improvement is fragile without structural alignment. The CNN's success on surface codes proves that architecture-code alignment is what makes pre-decoding robust. The MLP's partial success on BB codes shows the potential; closing the gap requires architectures that respect the code's actual connectivity.

Future Work

The results point to several promising directions:

  • CNN + MLP hybrids: BB codes have quasi-cyclic structure from ℤℓ × ℤm. Within each cyclic block, connectivity is regular (processable by small CNNs); inter-block long-range connections can be handled by MLP or attention layers. This mirrors how the surface code CNN exploits spatial regularity, but adapted for BB codes' block-cyclic topology.
  • Graph neural networks on the Tanner graph: GNNs operate directly on the check-qubit graph, learning message-passing rules that respect the actual connectivity [9] — a trainable BP with learned update functions. Quasi-cyclic weight sharing across ℤℓ × ℤm would reduce parameters and enable transfer across code sizes.
  • Learned symmetry breaking: The degeneracy problem is fundamentally about Tanner graph symmetry [2]. Neural networks can implicitly break these symmetries through nonlinear activations — the training loss drives commitment to specific corrections rather than degenerate oscillation.
  • Hardware-aware adaptation: Real devices have non-uniform noise. Combining neural pre-decoding with hardware-specific layout synthesis [5] and adaptive encoding [4] could yield end-to-end optimized systems calibrated to actual device noise profiles.

Just as the CNN pre-decoder succeeds by matching architecture to the surface code's spatial structure, future qLDPC pre-decoders should be designed from the code's Tanner graph outward.
